# The_Cyber_Workbench


## Cybersecurity Portfolio  
**By Anindya Sen**

Welcome to my cybersecurity portfolio repository. This is a personal hands-on project where I explored a wide range of cybersecurity, privacy, AI, and digital-safety topics through practical activities, research, mini implementations, and evidence collection. I built this repository as a structured record of what I learned, how I applied it, and what I observed while working through different real-world security-related tasks.

## About this repository

This repository brings together my work across multiple cybersecurity activities. Rather than only writing about concepts in theory, I focused on understanding how they appear in real life, how they can be analysed, and in some cases how small solutions or improvements can be designed and tested. The work includes a mix of written analysis, screenshots, presentation materials, code snippets, system observations, and tool-based evidence.

The topics covered in this repository include:

- **Security concepts** in everyday environments such as campus, public spaces, and home
- **Cryptography** used in websites, offline systems, modern networks, and IoT devices
- **Privacy techniques** used online and offline
- **Access control devices**, **locks**, and different forms of physical and digital protection
- **Offline and online security tools**
- **AI-enabled security solutions**
- **Recent security incidents** and local security incident analysis
- **Generative AI hallucination cases**
- **AI-generated image detection and verification**
- **Home cybersecurity improvements**
- **System bug research and fix logic**
- **Privacy-preserving design and implementation**
- **Security awareness communication**, including presentation and discussion tasks
- **Club/community engagement** in computing and cybersecurity-related spaces

## What you will find here

This repository is organised into activity-based folders. Each activity folder contains supporting material related to that specific task. Depending on the task, that may include:

- written notes and reflections
- screenshots and photographic evidence
- research links and references
- code snippets
- command outputs
- interface captures
- presentations or supporting documents
- metadata analysis
- implementation logic and design notes

Each folder is meant to show not only the final written outcome, but also the supporting evidence behind it.

## My approach

While building this portfolio, I tried to keep the work practical, detailed, and easy to follow. I was especially interested in connecting technical ideas with everyday situations, because many security and privacy issues become more meaningful when they are observed in real systems, devices, and user behaviour rather than only in theory.

Where possible, I used real tools, real interfaces, real outputs, and publicly available cases to support the work. In other places, I designed small solutions or implementation ideas to show how a problem could be handled better. My goal throughout this portfolio was to learn actively and document that learning clearly.

## Repository structure

The repository is divided into separate folders for different activities so that the work stays organised and easy to browse. A typical folder may contain:

- a short overview of the activity
- supporting screenshots or media
- any related code or commands used
- notes about findings, tools, or outputs
- extra evidence that supports the written report
