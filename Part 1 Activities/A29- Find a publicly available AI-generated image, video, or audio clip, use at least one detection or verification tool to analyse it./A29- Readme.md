## A29- Find a publicly available AI-generated image, video, or audio clip, use at least one detection or verification tool to analyse it.


For this activity, I chose to work with a publicly available AI-generated image taken from a BBC News article about AI-created photographs. I found this source useful because it was easy to access, openly discussed AI-generated imagery, and clearly stated that the images in the article were created using AI software on the NightCafe website. The article also explained that the people shown were not real and that the images had been generated from text prompts, which made it a suitable source for this task. I downloaded one of those AI-generated face images from the article and then analysed it using two separate AI-image detection websites so that I could compare the outputs rather than relying on only one result.

The first detection tool I used was Sightengine. Its result classified the uploaded image as “Likely AI-generated” with a very high overall score of 99%. What I found useful was that it did not only give one final label. It also broke the result down further and suggested which type of AI generation the image most likely resembled. In my result, the image was associated most strongly with Stable Diffusion, with smaller percentages for other systems. It also gave a very low score for face manipulation, which suggests the tool was treating the image as something generated synthetically rather than just an edited real photo. This made the output more informative and helped me see that these detectors often work using probability-style results rather than simple yes-or-no answers.

The second detection tool I used was Decopy AI, and its result was even stronger in a simpler way. It labelled the image as “likely AI-generated” with an AI-generated probability of 100% and a true image probability of 0%. What made this result interesting was the descriptive analysis it gave. Instead of only showing a number, it discussed things like skin texture and detail, eye realism, facial hair, lighting and shadow, and the background. Its report suggested that although the face looked realistic at first, some areas were still too uniform or too polished in a way that matched AI-generated portrait styles. I found this useful because it showed how AI-generated images can look convincing overall while still containing small visual clues that detectors notice.

After using the two detector websites, I also checked the metadata of the image in Kali Linux using the command exiftool. I compared the metadata of the AI-generated image with that of a non-AI image. This part was useful because it showed that image verification is not only about visual appearance or AI detectors, but can also involve looking at technical file information. In the AI image file, I noticed metadata fields such as “Producer: Online-Convert” and “XMP Toolkit: Image::ExifTool 12.65”, along with standard file details like file type, resolution, dimensions, and compression. In contrast, the non-AI PNG image showed a more ordinary set of image properties such as width, height, bit depth, colour type, gamma, and pixel units, but without the same kind of production-related tag. This does not automatically prove that one image is AI-generated and the other is not, because metadata can be changed, removed, or rewritten easily. However, it still gave me useful extra information about how the AI image may have been processed before I analysed it. In this case, the metadata suggests that the AI image had passed through another tool or conversion process, which is important because it reminds me that metadata alone is not enough to prove whether an image is AI-generated.

Overall, this activity helped me understand that verifying AI-generated media works best when using multiple methods together. The BBC source gave me a reliable origin for the AI image and confirmed that it was created from AI text prompts. Then the two detection tools, Sightengine and Decopy AI, both analysed the image and gave strong results pointing to it being AI-generated, while also giving useful detail about how they reached that conclusion. Finally, the metadata comparison in Kali Linux added another technical layer, showing that file properties can sometimes provide extra context, even if they are not enough on their own. What I found most useful from this task is that it is better to combine the source of the image, the detector outputs, and the metadata rather than trusting only one tool. This made the activity feel much more practical and realistic.
